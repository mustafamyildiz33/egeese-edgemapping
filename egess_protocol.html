EGESS

Experimental Gear for Evaluation of Swarm Systems — Protocol Overview

1. Core Idea

Every node periodically pulls its 6 hex neighbors with a small "are you there?" message. If a neighbor doesn't respond, it's missing. If a neighbor responds but reports much higher danger, that's disagreement.

The key insight: node unavailability itself is the sensing signal. By looking at which neighbors disappear, in what order, and whether the pattern continues or stalls, a node infers hazard approach, impact, containment, and recovery — without any environmental sensors.

2. What Each Pull Cycle Collects

SignalMeaningHow Detected
New missingNeighbor just went offline this cycleWas alive last cycle, pull failed now
Persistent missingWas already gone and still isWas missing last cycle, still missing
RecoveredWas gone but came backWas missing, pull succeeds now
DisagreementNeighbor sees higher danger than meNeighbor T ≥ 3 and T > myT + 1 (capped at 2)

3. Three Sub-Scores

Each node computes three scores from the same pull data. Each answers a different question:

FrontScore — "Is the hazard heading toward me?"
= 2 × newMissing + 1 × disagree + 0.5 × persistMissing + momentum
High when NEW neighbors disappear. Detects the approaching front. Disagreement capped at 2 max so it can't cascade across the whole grid.
ImpactScore — "How much damage is around me right now?"
= 3 × totalMissing + 2 × clusterBonus
Counts ALL missing neighbors. ClusterBonus = 1 if missing nodes are next to each other (structured hazard, not random).
ArrestScore — "Has the hazard stopped spreading?"
= stall + retreat − bypass
Stall = no new missing 3+ cycles. Retreat = neighbors recovering. Bypass = new damage in side direction. Positive = front stopped.

4. T Score

T = max(FrontScore, ImpactScore)
One number: "how bad is it?" Whichever sub-score is higher wins.
ModifierValueWhen
Momentum+2T was > 0 last cycle AND still getting signal
Decay−1No signal this cycle — T drops by 1 until 0

5. Slope — "Is it getting worse or better?" improved

Slope = (T_now − T_oldest) / (window − 1)
Over last 4 T values. Example: [0, 2, 5, 9] → slope = +3.0/cycle
T is a snapshot. Slope is the trend. Two nodes at T=3 can be in opposite situations: one approaching (slope +2), one recovering (slope −1).

Slope does two jobs in the state machine:

Rising → Early escalation

T ≥ 3, slope ≥ 2 → jump WATCH to WARNING

T ≥ 2, slope ≥ 1.5 → trigger WATCH early

Flat/declining → Containment

Stalled 3+ cycles, slope ≤ 0 → CONTAINED

Requires both temporal AND trajectory evidence

6. Coherence Gate — "Is this real or random?" improved

2 of 3 must be true before WATCH or WARNING triggers:

ConditionChecks
AdjacencyAre missing neighbors next to each other? Clustered = structured hazard.
PersistenceSame dominant direction for 2 of last 3 cycles? Consistent = real front.
ProgressionNew disappearances in that direction? Advancing = active front.

If only 1 of 3 → probably coincidence → don't escalate. This is the false alarm filter.

7. Protocol States

NORMAL
T = 0
All clear.
WATCH
T ≥ 3
Possible approaching hazard. Requires coherence ≥ 2.
WARNING
T ≥ 6
Confirmed front. Or T ≥ 3 with slope ≥ 2.
IMPACT
T ≥ 10
In the damage zone. No coherence needed.
CONTAINED
stall ≥ 3
Front stopped. No new damage + slope ≤ 0.
RECOVERING
rec > 0
Neighbors coming back. No new damage.
OFFLINE
—
Unreachable. Neighbors detect it as missing.
NORMAL → WATCH → WARNING → IMPACT → CONTAINED → RECOVERING → NORMAL

8. Spatial Hazard Reconstruction new

Each node's dominant sub-score reveals its role. Across the network, this reconstructs the hazard shape with no central coordinator:

LEADING EDGE

FrontScore highest — hazard heading this way next.

DAMAGE ZONE

ImpactScore highest — surrounded by offline neighbors.

CONTAINMENT BOUNDARY

ArrestScore dominant — the front stopped here.

UNAFFECTED

No elevated scores. Normal operations.

How to Run

9. Start the Node Network original

Start 64 nodes on an 8×8 hex grid:

# Start all 64 nodes in background
for i in $(seq 0 63); do python3 node.py $((9000+i)) 64 & done

Stop all nodes:

# Kill all running node processes
pkill -f "python3 node.py"

10. Manual Node Control original

Kill a node (simulate crash):

curl -s -X POST http://127.0.0.1:9010/ \
  -H "Content-Type: application/json" \
  -d '{"op":"inject_fault","data":{"fault":"crash_sim","enable":true}}'

Bring a node back:

curl -s -X POST http://127.0.0.1:9010/ \
  -H "Content-Type: application/json" \
  -d '{"op":"inject_fault","data":{"fault":"crash_sim","enable":false}}'

Set a node to ALERT (simulate sensor detection):

curl -s -X POST http://127.0.0.1:9010/ \
  -H "Content-Type: application/json" \
  -d '{"op":"inject_state","data":{"sensor_state":"ALERT"}}'

Set a node to RECOVERING:

curl -s -X POST http://127.0.0.1:9010/ \
  -H "Content-Type: application/json" \
  -d '{"op":"inject_state","data":{"sensor_state":"RECOVERING"}}'

Reset a node to NORMAL:

curl -s -X POST http://127.0.0.1:9010/ \
  -H "Content-Type: application/json" \
  -d '{"op":"inject_state","data":{"sensor_state":"NORMAL"}}'

11. Pull a Node's State original

Pull full state from a single node (shows T, scores, neighbors, everything):

curl -s -X POST http://127.0.0.1:9018/ \
  -H "Content-Type: application/json" \
  -d '{"op":"pull","data":{"kind":"neighbor_status"},"metadata":{"origin":"debug"}}' \
  | python3 -m json.tool

12. Original Matplotlib Visualizer original

Run the dual-panel matplotlib visualizer (needs running nodes):

# Gossip View + Hex Map, interactive click-to-inspect
python3 visualize_mapping.py --base-port 9000 --n 64 --fps 2

Run with auto-demo (corner spread):

python3 visualize_mapping.py --base-port 9000 --n 64 --auto-demo spread --auto-period 10

Run with auto-demo (tornado):

python3 visualize_mapping.py --base-port 9000 --n 64 --auto-demo tornado --auto-period 10

13. Terminal Traffic Monitor improved

Live-updating terminal table showing all node states, T scores, sub-scores, slope, pull/push traffic counts, score breakdowns, and recent message traffic. Color-coded ANSI output. Use 16 nodes by default so it fits on a Mac screen more comfortably.

Monitor only (watch live traffic):

python3 egess_monitor.py --n 16 --base 9000 --refresh 2

Three-terminal setup for a small Mac-friendly run:

# Terminal 1: start 16 real nodes (no visualizer)
cd /Users/mustafa/egess
./startnodes --nodes 16 --no-visualizer

# Terminal 2: run the full monitor (shows all 16 nodes)
cd /Users/mustafa/egess
python3 egess_monitor.py --n 16

# Terminal 3: stop everything when you're done
cd /Users/mustafa/egess
./stopnodes

Monitor with compact view (only show active nodes):

python3 egess_monitor.py --n 16 --compact

Corner spread demo (show all 16 node states):

python3 egess_monitor.py --demo spread --n 16

Important: if you want to see every node in the table, do not use --compact. Compact mode hides quiet nodes and only shows active threat rows.
